C · weir wire client

A dependency-free C producer that speaks the weir v1 wire protocol over a Unix socket — built from the spec, zero weir-crate deps, just the C standard library and POSIX sockets. The use case: embedded-systems telemetry.

28/28 conformance vectors byte-exact + 109 checks C stdlib + POSIX zero Rust deps
What it is

A dependency-free C producer for the weir daemon, built entirely from docs/wire_protocol.md and the conformance vectors — no weir crate is linked. A zero-dependency wire codec (weir_wire.c), a blocking POSIX AF_UNIX transport with correct stream framing (weir_conn.c), and a telemetry producer (telemetry.c).

The use case is embedded telemetry: a sensor node emitting compact fixed-width binary records — the kind of payload an MCU actually pushes, not JSON — and streaming them to weir for durable buffering.

Run it
Conformance no daemon needed
make            # builds: conformance, telemetry, negative
./conformance /path/to/weir/docs/conformance/wire_v1_vectors.json

Expected: vectors scanned: 28   checks passed: 109   failed: 0 → C codec is wire-compatible with weir v1. The strongest check is byte-exact re-encoding.

Requirements: C11 + POSIX sockets, no deps · requires a C11 compiler (-std=c11) · tested with clang 21.

The codec
weir_wire.c Push encode — 16-byte header + payload + CRC32
weir_result weir_encode_push(weir_durability dur,
                             const uint8_t *payload, size_t payload_len,
                             uint8_t *out, size_t out_cap, size_t *out_len) {
    if (payload_len == 0) return WEIR_ERR_EMPTY_PAYLOAD;
    if (payload_len > WEIR_MAX_PAYLOAD_HARD_CAP) return WEIR_ERR_PAYLOAD_TOO_LARGE;

    size_t total = WEIR_HEADER_LEN + payload_len + WEIR_CRC_LEN;
    if (out_cap < total) return WEIR_ERR_BUF_TOO_SMALL;

    write_header(out, WEIR_MSG_PUSH, dur, (uint32_t)payload_len);
    memcpy(out + WEIR_HEADER_LEN, payload, payload_len);
    put_u32_le(out + WEIR_HEADER_LEN + payload_len,
               weir_crc32(payload, payload_len));
    if (out_len) *out_len = total;
    return WEIR_OK;
}

CRC is CRC-32 / ISO-3309 (zlib / PNG / Ethernet) — a lazily-built 256-entry reflected table, not CRC-32C. The tree builds warning-clean under strict flags.
