TypeScript · weir wire client

An async HTTP backend's durable request log. A dependency-free TypeScript/Node producer that speaks the weir v1 wire protocol over a Unix socket — built from the spec, zero weir-crate deps, no npm runtime deps, Node stdlib only (node:net, node:zlib) — durably logging each incoming request to weir before the handler replies.

What it is

A dependency-free TypeScript producer built purely from docs/wire_protocol.md and the conformance vectors — no weir crate, no npm runtime deps. A reference codec (wire.ts) and an async WeirClient (client.ts) doing serial request/response over one connection.

It demonstrates a realistic async-integrator use case: a Node http web backend that durably logs every request to weir from its handler. Runs .ts directly on stock Node — no build step.

The codec
src/wire.ts frame encode — 16-byte header + payload + CRC32
// Encode a frame. Defaults to a Push at Sync durability.
export function encodeFrame(payload: Buffer, opts: EncodeOpts = {}): Buffer {
  const messageType = opts.messageType ?? MessageType.Push;
  const durability = opts.durability ?? Durability.Sync;
  const flags = opts.flags ?? 0;

  const frame = Buffer.allocUnsafe(HEADER_LEN + payload.length + 4);
  MAGIC.copy(frame, 0);                          // "WEIR"
  frame.writeUInt8(WIRE_VERSION, 4);
  frame.writeUInt8(messageType, 5);
  frame.writeUInt8(durability, 6);
  frame.writeUInt8(flags, 7);                    // reserved, must be 0
  frame.writeUInt32LE(payload.length, 8);
  frame.writeUInt32LE(crc(frame.subarray(0, 12)), 12); // header CRC
  payload.copy(frame, HEADER_LEN);
  frame.writeUInt32LE(crc(payload), HEADER_LEN + payload.length);
  return frame;
}

CRC is node:zlib.crc32 (IEEE / ISO-3309), not CRC-32C — it matched every worked-example CRC on the first try.
