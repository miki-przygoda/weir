Java · weir wire client

An enterprise audit-event logger. A self-contained JDK producer that speaks the weir v1 wire protocol over a Unix domain socket — built from the spec, zero weir-crate deps, only the JDK standard library (no Maven, no Gradle, no third-party jars) — writing each audit event at Sync durability, so the producer only proceeds once the record is fsynced.

29/29 conformance vectors byte-exact JDK stdlib only zero Rust deps
What it is

A clean polyglot Java implementation of the wire, written straight from docs/wire_protocol.md with no dependency on any weir crate — exactly what a skeptical enterprise team would write to integrate a JVM service with weir without pulling in a Rust toolchain. It uses UnixDomainSocketAddress + SocketChannel (JDK 21+) for the transport and java.util.zip.CRC32 for the CRC.

The domain is enterprise event logging: AuditEventProducer streams structured JSON audit events to weir at Sync durability — so an Ack means "on stable storage", the guarantee a compliance auditor cares about.

The codec
Frame.java frame encode — 16-byte header + payload + CRC32
// Encodes this frame: HEADER_LEN + payload.length + CRC_LEN bytes.
public byte[] encode() {
    int total = Wire.HEADER_LEN + payload.length + Wire.CRC_LEN;
    ByteBuffer buf = ByteBuffer.allocate(total).order(ByteOrder.LITTLE_ENDIAN);

    buf.put(Wire.MAGIC);                   // [0..4)  "WEIR"
    buf.put((byte) Wire.WIRE_VERSION);     // [4]
    buf.put(messageType.code);             // [5]
    buf.put(durability.code);              // [6]
    buf.put((byte) (flags & 0xFF));        // [7]  reserved, must be 0
    buf.putInt(payload.length);            // [8..12) LE
    buf.putInt((int) crc32(buf.array(), 0, 12)); // header CRC over [0..12)
    buf.put(payload);                      // [16..16+n)
    buf.putInt((int) crc32(payload, 0, payload.length));
    return buf.array();
}

CRC is java.util.zip.CRC32 (IEEE / ISO-3309), not CRC-32C. Built and verified on OpenJDK 25; requires JDK 21+ (current LTS).
