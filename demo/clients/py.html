Python · weir wire client

A pure-Python producer that speaks the weir v1 wire protocol straight to the daemon socket — built from the spec, zero weir-crate deps, stdlib only (socket, struct, zlib).

28/28 conformance vectors byte-exact stdlib only zero Rust deps
What it is

A from-scratch, stdlib-only Python implementation of the weir v1 wire protocol — no Rust, no weir crates imported. It talks to the daemon over its AF_UNIX socket using the framing defined in docs/wire_protocol.md: a frame codec (codec.py) and a synchronous WeirClient with push() and health_check().

It was built the way a polyglot developer would — working only from docs/wire_protocol.md and the conformance vectors, never looking at the Rust source or the shipped reference codec. A working client fell out in one pass.

The codec
src/weir_wire/codec.py frame encode — 16-byte header + payload + CRC32
def encode_frame(message_type, durability, payload, flags=0):
    """Encode exactly one frame. Mirrors the wire layout table."""
    header = bytearray(HEADER_LEN)
    header[0:4] = MAGIC                # b"WEIR"
    header[4] = WIRE_VERSION             # 1
    header[5] = int(message_type)
    header[6] = int(durability)
    header[7] = flags                   # reserved, must be 0
    struct.pack_into("<I", header, 8, len(payload))
    struct.pack_into("<I", header, 12, _crc32(bytes(header[:12])))
    return bytes(header) + payload + struct.pack("<I", _crc32(payload))

CRC is IEEE / ISO-3309 CRC-32 (Python's zlib.crc32), not CRC-32C. The encoder reproduces the spec's worked examples byte-for-byte.
