Go · weir wire client

The adversarial conformance harness. A pure-Go producer that speaks the weir v1 wire protocol straight to the daemon socket — built from the spec, zero weir-crate deps, stdlib only (hash/crc32, encoding/binary, net) — paired with a 15-case live harness that deliberately fires every rejection (Nack) reason at a running daemon and checks it gets the right one back.

What it is

A from-scratch Go implementation of the weir v1 wire protocol, built only from docs/wire_protocol.md and the conformance vectors — it does not link the Rust weir-client. A frame codec (wire.go), a synchronous producer over a Unix socket (client.go), and an adversarial live harness (main.go).

The goal: prove the wire spec is sufficient to build a wire-compatible producer in another language, and adversarially test every Nack reason and edge case against a live daemon.

The codec
wire.go frame encode — 16-byte header + payload + CRC32
// EncodeFrame builds a complete on-the-wire frame.
func EncodeFrame(f Frame) []byte {
    plen := len(f.Payload)
    buf := make([]byte, HeaderLen+plen+4)
    copy(buf[0:4], magic[:])              // "WEIR"
    buf[4] = f.Version
    buf[5] = byte(f.MessageType)
    buf[6] = byte(f.Durability)
    buf[7] = f.Flags                      // reserved, must be 0
    binary.LittleEndian.PutUint32(buf[8:12], uint32(plen))
    binary.LittleEndian.PutUint32(buf[12:16], crc(buf[0:12]))
    copy(buf[16:16+plen], f.Payload)
    binary.LittleEndian.PutUint32(buf[16+plen:], crc(f.Payload))
    return buf
}

crc is the IEEE / ISO-3309 CRC-32 (crc32.IEEETable), not CRC-32C.
