weir live demo

A durable, high-throughput write buffer — pushing records the fast, safe way.

What you're looking at. Producers fire records at weir over a local socket. weir writes each to a CRC-checked, append-only Write-Ahead Buffer (WAB), fsyncs according to the durability tier, acks the producer, then asynchronously drains batches to the downstream database — N records collapse into 1 commit. The promise: an ack is never a false ack. An acked record is durable and will be delivered, even across a crash. Push some records, then hit Crash daemon and Restart to watch unconfirmed segments replay. The top lane is a naive insert-per-record baseline for comparison. This is a browser simulation; latencies are modeled on weir's measured benchmarks.

How faithful is this to real weir?

The motion is illustrative, but the model follows weir's real semantics:

  • Sync — fsync before the ack. The producer blocks until the record is on stable storage. Zero acked-record loss on crash.
  • Batched — the record is written before the ack, and one group fsync covers the whole batch. Still durable before the ack, but N records share one fsync — the throughput sweet spot.
  • Buffered — the ack is sent after the in-memory enqueue only. Fastest, but a crash can lose the records in the not-yet-fsynced window. The demo shows those as lost — weir is honest that Buffered trades durability for latency.
  • Drain & commit compression — the drain reads a sealed WAB segment and commits the whole batch downstream in one statement (the SQL sinks do N rows → 1 INSERT). Segments are not reclaimed until the sink confirms.
  • Crash recovery — on restart, weir replays every sealed-but-unconfirmed segment, so anything durably written (Sync/Batched) is redelivered. That is the ack is never a false ack guarantee.

Latency figures are rounded from the project's CI benchmarks (Sync/Batched ≈ 0.39 ms ack, Buffered ≈ 0.07 ms). The naive baseline models a synchronous insert + commit + network round-trip to a remote database (~8 ms), one commit per record.

Browser-only simulation of weir. Real weir speaks a 16-byte-header wire protocol over a Unix socket (or TCP + mutual TLS).
