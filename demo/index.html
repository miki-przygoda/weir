A durable write buffer — same speed as a direct insert, but crash-safe and N→1 on the database.

The pitch. A "usual" setup inserts each record straight into the database and blocks the producer until it commits. weir instead takes the record over a local socket, writes it to a CRC-checked Write-Ahead Buffer, fsyncs per the durability tier, and acks the producer — then drains batches downstream, collapsing N commits into one. You get two things the direct path can't: an ack is never a false ack (an acked record is durable and replays across a crash) and far fewer database commits — at a producer-facing latency that's as fast or faster than the direct insert. The honest tradeoff: a record lands in the database a beat later, via the batched drain. Push records below and compare the two stacks. How faithful is this to real weir?

The motion is illustrative, but the model follows weir's real semantics, and per-record latencies are drawn from a long-tailed distribution (hence the avg / p99 / max):

  • The first hop is the same in both lanes — getting the record off the producer. The stacks diverge after it: weir does a fast local fsync and acks (~0.4 ms Sync); the naive path blocks on the remote commit round-trip (~8 ms). Same database, so weir's drain commit costs the same as a naive commit — but one trip carries the whole batch.
  • Sync — fsync before the ack; zero acked-record loss on crash.
  • Batched — written before the ack, one group fsync per batch; still durable before the ack, far fewer fsyncs.
  • Buffered — ack after the in-memory write only; fastest, but a crash can lose the not-yet-fsynced window (shown as lost — weir is honest about the tradeoff).
  • Crash recovery — on restart weir replays every sealed-but-unconfirmed segment, so anything durably acked is redelivered. The naive lane talks straight to the DB and is independent of the weir daemon.
  • End-to-end — a record reaches the database later under weir (it waits to be batched). weir trades end-to-end freshness for fast durable acks, far fewer commits, and producer/​downstream decoupling — not for raw insert speed.

Browser-only simulation of weir. Real weir speaks a 16-byte-header wire protocol over a Unix socket (or TCP + mutual TLS).
