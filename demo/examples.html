The recipes

Minimal producer

weir-client

The smallest useful thing: connect to the socket and push a durable record. One crate, no async runtime, one blocking round-trip — the call returns only once the record is durable.

use weir_client::{WeirClient, Durability};

let mut client = WeirClient::connect("/run/weir/weir.sock")?;
client.push(b"hello, weir!", Durability::Sync)?;  // durable before this returns

Producer in an async runtime

weir-client+ your runtime

push() is blocking — calling it in an async fn starves the executor. The bridge: a pool of dedicated threads, each owning one persistent client, fed by a bounded channel; the async side awaits a reply. Reconnect on poison via the client API (no matching the #[non_exhaustive] error enum).

// one std::thread owns a persistent client; the async side awaits a oneshot
let (tx, rx) = std::sync::mpsc::sync_channel::<Job>(1024);
std::thread::spawn(move || {
    let mut client = WeirClient::connect(&sock).unwrap();
    for job in rx {                              // Job { payload, durability, reply }
        let r = client.push(&job.payload, job.durability);
        if client.is_poisoned() { client = reconnect(&sock); }
        let _ = job.reply.send(r);                   // tokio oneshot back to the caller
    }
});

Custom sink

weir-sink-sdkweir-core

Forward records to your own destination. Implement Sink against a frozen contract — and unit-test it with no daemon and no runtime. Each record ends up committed or dead-lettered (permanent rejection); a transient failure returns Err and the whole segment is retried.

use weir_sink_sdk::{Sink, CommitResult, Payload, BasicSinkError, SinkHealth};

impl Sink for MySink {
    type Record = Payload;
    type Error  = BasicSinkError;
    async fn commit(&self, batch: Vec<Payload>)
        -> Result<CommitResult<Payload>, BasicSinkError> {
        let mut committed = Vec::new();
        let mut dead = Vec::new();
        for rec in batch {
            match deliver(rec.as_ref()) {
                Ok(()) => committed.push(rec),
                Err(why) => dead.push((rec, why)),   // permanent → dead-letter
            }
        }
        Ok(CommitResult::new(committed, dead))
    }
    async fn health(&self) -> SinkHealth { SinkHealth::Healthy }
}

Offline recovery / export tool

weir-wab

Read the on-disk buffer directly — for disaster recovery, export, or inspection — with a runtime-free reader and a tiny dep tree. SegmentReader streams each record and verifies its CRC32, stopping at the first corruption with a clean error.

use weir_wab::{SegmentReader, Payload};   // Payload re-exported — no weir-core dep needed

for rec in SegmentReader::open("/var/lib/weir/wab/shard_00/seg_00000001.wab")? {
    let payload: Payload = rec?;       // io::Error on a CRC mismatch / truncation
    // … inspect, re-emit as NDJSON, or recover …
}

Admin & inspection

weir-ctl

Operate a running daemon from the shell — no code, no daemon dependency tree. Live commands hit the socket / metrics endpoint; segments and dl read the WAB on disk. dl drop/requeue default to a dry run.

weir-ctl health   --socket /run/weir/weir.sock
weir-ctl metrics  --addr 127.0.0.1:9185
weir-ctl segments --wab-dir /var/lib/weir/wab        # per-shard WAB inspect
weir-ctl dl list  --wab-dir /var/lib/weir/wab        # dead-letter queue
weir-ctl dl requeue --wab-dir /var/lib/weir/wab --yes # re-submit (dry-run without --yes)

Polyglot / non-Rust client

wire protocol onlyzero Rust deps

Produce from any language by speaking the wire protocol directly — a 16-byte header + payload + CRC32 over a Unix socket (or TCP + mutual TLS). Language-neutral conformance vectors pin every byte, so you can prove byte-compatibility.

# 16-byte header, then payload, then payload CRC32 (IEEE/zlib):
#   magic "WEIR" · version · msg_type(Push) · durability · flags
#   · payload_len (u32 LE) · header-CRC32   →   payload bytes   →   payload-CRC32

docs/conformance/wire_v1_vectors.json     # 28 canonical vectors
python docs/conformance/run_vectors.py    # reference runner — 28/28

Full self-hosted pipeline

weir-serverweir-client+ a sink

The whole thing: run the daemon with a real sink, point producers at the socket, and let the drain compress N records into one downstream commit. Crash-safe by construction — unconfirmed segments replay on restart.

weir-server --wab-dir /var/lib/weir/wab \
            --socket-path /run/weir/weir.sock \
            --sink-type http --sink-url https://ingest.example/bulk

# producers push via weir-client over the socket;
# the drain batches and forwards N records → 1 commit to the sink.
