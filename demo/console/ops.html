Ops · clearing a dead-letter backlog

Your downstream sink — a warehouse, an HTTP endpoint — was down for an hour. weir kept its promise: it accepted and acked every record (durable on disk) and set aside what the sink rejected into the dead-letter store. The sink is healthy again. Now you decide what happens to the backlog.

preview → confirm → result requeue = re-push through the daemon drop = permanent delete
The scenario

Dead-lettering is weir's release valve: a record that the sink permanently rejects (or that's left stranded when the sink is unreachable) is moved aside so the pipeline keeps flowing instead of wedging. Nothing is lost — it's all still on disk — but it needs a human decision once the downstream recovers.

What you're looking at

Status header (top) — a live read of the daemon: up/down, sink health + type, accepted/ack/nack, queue depth (drain backlog), average fsync, WAB + dead-letter bytes, and panic/fsync-failure alarms.

Requeue all — re-submits every dead-lettered record back through the daemon's socket (not a file move). It shows a dry-run preview first (records, segments, any unreadable ones it'll skip), lets you choose a durability tier, then commits. Re-delivery is at-least-once — identical payloads are deduped by the sink's idempotency key.

Drop all — permanently deletes the segments. It previews the count, then makes you type the segment count to confirm. (When the daemon looks live, both actions add one more confirmation, since you're touching a store the daemon may still be writing to.)

Try Requeue all or Drop all above — the sample store empties exactly as the real one would.

Why it matters

Operating dead-letters is the dangerous part of running a buffer — get it wrong and you double-deliver or lose data. The Ops panel wraps it in preview-then-confirm and, under the hood, shells out to the same tested weir-ctl the CLI uses. The web buttons can't drift from the command line, and a --read-only mode disables every mutation for shared instances.

The real tool

This page is a sample-data mock (no backend). Ops needs a running daemon — for the status header and for requeue's re-push:

cargo run --manifest-path tools/weir-console/Cargo.toml -- \
  --wab-dir <dir> --metrics-addr 127.0.0.1:9185 --socket /run/weir/weir.sock
# open the Ops tab; --read-only disables requeue/drop for shared instances

weir-console — one local tool, three views (Explorer · Ops · Live). Ops mutations reuse weir-ctl; nothing here re-implements the durability-critical paths. Full reference at the docs site.
